## 3. VISUAL IDENTITY

### 3.1 Color Palette

| Token name | Hex | RGB | Use case |
|---|---|---|---|
| **Soft Sandstone** | `#F5F5F5` | 245, 245, 245 | Primary body backgrounds, light surfaces, breathing room |
| **Cosmic Navy** | `#1A1A2E` | 26, 26, 46 | Hero and footer dark sections, premium signal, authority surfaces |
| **Electric Cyan** | `#00E0FF` | 0, 224, 255 | CTAs, accent lines, hover states, attention focal points |
| **Muted Gold** | `#D4AF37` | 212, 175, 55 | Premium tier indicators, italic taglines, exclusive elements |
| **Subtle Red** | `#C73E3A` | 199, 62, 58 | Negative states, X icons, anti-patterns marking |

**Usage rules:**
- Cosmic Navy and Soft Sandstone dominate. ~80% of any composition.
- Electric Cyan = 1-2 accent uses per composition, never as fill.
- Muted Gold = reserved for premium signals (whale-tier offers, AGS founder voice flourishes).
- Subtle Red = error states or "do not do this" indicators only. Never for emphasis.
- No gradients between palette colors. No tints/shades outside this palette.

### 3.2 Typography

| Font | Weights | Use case |
|---|---|---|
| **Playfair Display** | Bold, SemiBold, Italic | All headlines, large quotes, taglines, decorative emphasis |
| **DM Sans** | 400 (Regular), 500 (Medium) | Body text, paragraphs, descriptions, captions |
| **Inter** | 400, 600 (SemiBold) | UI elements, buttons, small text, navigation, fallback for DM Sans |
| **JetBrains Mono** (added v1.1) | 400, 500, 700 | Code snippets, technical badges, category mono chips, system signal labels, engineering-context emphasis |

**Pairing rules:**
- Headlines always Playfair Display. Body always DM Sans. UI always Inter. Code or technical context always JetBrains Mono.
- Never use Playfair for body (illegibility at small sizes).
- Never use DM Sans or Inter for primary headlines (too generic).
- Tagline display: Playfair Display Bold Italic 22-26px.
- Numbers in copy ARE anchors (37 faxes, 140zł, 200K). Use Playfair Display Bold or Inter SemiBold for them, oversize when alone.

### 3.3 Logo

**Status:** wordmark only as of 17/05/2026. Logomark TBD.

**Wordmark form:**
- Full: "Authentic Growth Systems" in Inter SemiBold or Playfair Display SemiBold depending on context.
- Abbreviated: "AGS" in same fonts.

**Placement rules:**
- Bottom right in covers and slides (60% opacity if non-primary element).
- Top left in web headers.
- Never centered (centered logo signals static brand, AGS is in motion).

**Clear space rule (added v1.1, Tomasz directive):**
- Minimum clear space around logo, wordmark, and G monogram watermark = approximately 4-5% of canvas dimension on every side.
- Logo must breathe within the composition, never press against canvas edges.
- Same rule applies to G monogram watermarks placed within compositions.
- Violation = drift toward cramped corporate aesthetic, away from AGS engineering precision.

**G monogram (added v1.1):** Accepted as logomark companion to wordmark. Used as social avatar (LinkedIn profile pic, X profile pic, Notion icon), seal element ("G on gold" variant for premium signal), and watermark within compositions. Three canonical variants: outline G monogram on light, white G on Cosmic Navy ink, white G on Muted Gold seal.

**G monogram watermark size + overlap rule (added v1.1, Tomasz directive, refined three times):**
- When G monogram is used as watermark within a composition, it must be LARGE, occupying 60-70% of canvas dimension. Treat as dominant signature element grounding the composition, not corner accent.
- Watermark MUST STAY FULLY WITHIN CANVAS. Large but UNCROPPED. The complete monogram shape must be visible. No bleeding off any edge.
- Minimum 4-5% canvas dimension clear space from ALL edges (top, bottom, left, right).
- Anchor position: typically bottom-right corner anchor for signature feel. Composition reads: text left half, monogram bottom-right anchor, circuit motif fills upper/middle band.
- SYMMETRICAL ANCHOR rule: when monogram is bottom-right anchored, clear space from BOTTOM edge MUST equal clear space from RIGHT edge. Symmetrical breathing room on both edges, not skewed toward one. Visual balance.
- Clear space tightening for signature anchor: when monogram functions as dominant signature element in bottom-right anchor position, clear space from bottom and right edges may be tightened to 2-3% canvas dimension (instead of default 4-5%). Symmetrical equal on both edges maintained.
- Vertical centering exception: when monogram lives in a dedicated split panel (variant D pattern), it MUST be vertically centered between the panel's top edge and any bottom-anchor decorative element (circuit trace, etc.), with equal space above and below.
- Watermark MUST NOT overlap or visually interfere with text elements (headline, body, eyebrow, byline, AGS wordmark, reading time, slide number).
- Use opacity 5-15% so monogram decorates background without competing with text readability.
- If monogram does not fit fully within canvas at large size with proper clear space and without overlapping text, use the variant D (split case study) logic: dedicate a portion of canvas as separate panel where the full monogram lives with breathing room.
- Reference: cover variant D (split case study) is the canonical reference for dedicated-panel monogram placement.

**Circuit motif (added v1.1):** Accepted as signature element. Thin line "trace + terminal pin" geometric pattern referencing Tomasz's Electronics and Telecommunications engineering background. Used as section dividers, footer ornaments, subtle backgrounds at 5-10% opacity. Gold terminal pin marks trace completion ("gold terminates the trace").

**Circuit motif overlap rule (added v1.1, Tomasz directive):**
- Circuit motif (traces + terminal pins) MUST NOT overlap text elements.
- Traces and text must occupy separate visual zones, typically opposite corners or distinct bands of the composition.
- When circuit motif and text would visually compete, reposition or scale down the circuit element.

**Circuit motif placement rule (added v1.1, Tomasz directive, refined):**
- Circuit traces work as subtle background element or section bridge, NOT as floating ornament alongside text or visuals.
- On web hero section: place circuit trace at the very BOTTOM of the light hero as horizontal bridge into the next dark section.
- On product visuals: place circuit trace BEHIND or UNDER the visual element, as background, never beside it competing for attention.
- Opacity for background circuit motif: 5-10% (subtle, decorative, never demanding).
- Do not place a horizontal trace line floating above or beside content blocks without anchoring purpose.

**Circuit motif size rule (added v1.1, Tomasz directive, refined):**
- When circuit trace is used as section accent or bottom bridge, target 50-70% of canvas width (or section width). When used to fill negative space alongside a large bottom-right anchored monogram, scale closer to 70%.
- Small fragmentary trace (20-25% width) looks like floating ornament, not signature element. Avoid.
- Maintain 4-5% clear space from all text elements (no touching wordmark, byline, headlines, reading time, url).
- Position circuit trace anchors a band of the composition (typically upper or middle band when monogram lives bottom-right; typically bottom band when monogram is absent or small).
- Opacity 5-10% when filling negative space, so it adds engineering signature texture without competing with text or large monogram.

### 3.4.1 Photo of Tomasz (founder photography, added v1.1, refined v1.2)

Real photographs of Tomasz are a primary brand asset. Founder face = build-in-public credibility, peer-to-peer authority. Frequently featured on covers, hero sections, founder-voice content.

**CORRECTION from earlier draft (Tomasz directive 17/05):** Photo on covers is treated as FULL CANVAS BACKGROUND with text overlaid on it, NOT as a side panel with text in separate zone. This matches Tomasz's existing production style.

**Photo treatment:**
- Full color preferred (vs B&W or duotone) for authenticity.
- Background behind subject should be Cosmic Navy or dark tonal range for text readability when overlay applied. No busy environments, no clutter, no corporate office stock.
- Real working contexts OK if intentional (dance floor, computer visible, build-in-public moment).
- No heavy filters, no Instagram aesthetics, no glamour shots. Engineer precision + artist soul = honest portraiture.

**Photo placement on covers (FULL BLEED, primary approach):**
- Photo fills ENTIRE canvas as background layer.
- Subject (Tomasz) occupies upper 60-70% of canvas (head and shoulders), darker zone at bottom 30-40% reserved for text overlay readability.
- Crop: head-and-shoulders portrait (most common), or upper body in active build context.
- Use subtle opacity overlay or vignette if needed on darker zone to ensure white text reads cleanly over photo.

**Text overlay on photo:**
- Text OVERLAID on photo in same canvas zone (not in separate panel).
- Headline font: **DM Sans Bold or Inter Bold** (sans-serif for legibility on photo backgrounds). **NOT Playfair Display for photo covers** (serif fights with imagery, sans wins at impact + readability).
- Headline size: 60-80pt on portrait canvases.
- Headline color: white primary. One or two emphasis words allowed in Muted Gold OR Electric Cyan italic.
- Sub-headline: smaller (28-32pt), often in Electric Cyan for accent.
- Optional: Muted Gold horizontal accent line with arrow below sub-headline.

**Brand composition elements on photo cover:**
- G monogram TOP-RIGHT in Muted Gold, CLEAN (no glow, no sparkle, no shine star effect). Just the monogram shape.
- Decorative dotty pattern TOP-LEFT (geometric ornament accent).
- Bottom-left: small circular AVATAR PHOTO of Tomasz (head shot, circular crop) with Muted Gold border. NOT the G monogram in this position, use real founder face. Plus name "Tomasz Nawrocki" + one-line tagline beside the avatar.
- Bottom-right: action/signature icon (e.g. repost icon for repost content, custom icon for context).

**Constraints:**
- 4-5% clear space rule still applies for top corners (G monogram, dotty pattern).
- Photo must not be cropped tightly to face only (always include shoulders or context).
- One photo per composition.
- Photo edges MUST bleed seamlessly into the dark background. NO visible border, frame, outline, or edge separation between photo and canvas. The transition between photo and Cosmic Navy background is invisible.
- Footer text (wordmark, reading time, domain url, attribution) MUST be rendered as text-only on the canvas. NO background box, NO panel, NO shadow behind footer text. Text reads directly over the dark background or photo area.

**Use cases:**
- LinkedIn carousel slide 01 cover variant (photo).
- LinkedIn Article cover photo variant (same full-bleed approach adapted to 1280x720 16:9).
- Web /about hero section (large founder face section).
- Sales deck founder intro slide.
- Podcast episode covers, guest speaker assets.

**Anti-patterns:**
- Photo as side panel with text in separate zone (this was earlier draft direction — REJECTED, use full bleed instead).
- Photo as background watermark behind text in small format (photo is too information-dense for watermark function, use G monogram instead).
- Multiple Tomasz photos on same cover (one photo, one composition).
- Photo cropped tightly to face only.
- Photo with text overlaid directly on the face (text must occupy darker zone away from face).
- Photo cover using Playfair Display for headline (serif loses to sans-serif on photo backgrounds).

### 3.5 Layout Principles

**Three-layer visual logic:**
1. **Bold typographic hierarchy.** Headline dominates. Subhead supports. Body explains.
2. **Clean geometric composition.** Rule of thirds, asymmetric balance, generous whitespace.
3. **Restrained color accents.** Most surface in Cosmic Navy or Soft Sandstone. Cyan or Gold as 1-2 focal points.

**Premium depth rules (added v1.1, Tomasz directive):**
- Flat single-color sections feel cheap. Add subtle vertical gradients WITHIN canonical palette tones only (e.g. Cosmic Navy top to slightly deeper navy bottom). Never introduce non-canonical colors.
- Card elevations on dark backgrounds: slightly lighter navy fill plus very soft drop shadow so cards "lift" off the section background.
- Gold-filled CTA buttons: subtle Muted Gold to slightly darker Muted Gold gradient for premium feel.
- Gradients must be tasteful and restrained, never glossy, never web-2.0 reflective.

**Multi-card alignment rule (added v1.1):**
- When three or more cards sit side by side in a row, they MUST be vertically aligned in height. Card boxes match top and bottom, body copy length varies but containers do not.
- Forced equal-height alignment prevents visual chaos.

**Pull quote section alignment (added v1.1, Tomasz directive):**
- Pull quote sections (canonical "Authentic is not a vibe..." or similar Tagline Library quotes) center quote text, attribution, and quote mark horizontally on the canvas.
- Pull quote = breathing pause in page rhythm. Center alignment reinforces that pause.

**Two-column section vertical alignment (added v1.1):**
- When section uses two-column layout (visual on one side, text on other), columns must be vertically center-aligned to each other. Visual element must not sit higher or lower than text column without intentional reason.

## 5. FORBIDDEN PATTERNS

### 5.1 Voice-related forbidden (per AGS_CANONICAL_RULES.md and SALES_BIBLE.md PART 4)
- Em dash character `—` anywhere. Always.
- Banned phrases: "unlock your potential", "game-changer", "leverage", "synergy", "disrupt", "thought leader", "crush it", "ecosystem", "friction", "acquisition model", "optimize", "curious" (overuse), "in today's fast-paced AI landscape" and similar generic openers.
- Never claim AGS has paying clients (Stage 0-1 reality).
- Never claim specific client counts ("Currently working with 11 founders", "Trusted by 50+ founders"). Includes implied counts.
- Never write fake scarcity claims ("Limited to 4 founders per quarter", "Only 3 spots left this month") when scarcity is not real. Stage 0-1 = no real scarcity to limit.
- Never claim specific ROI results without verifiable test data.
- Never reference `/apply` page (does not exist).
- Never start DMs with "Glad we're connected!".
- Never mention Calendly (manual DM scheduling for now).
- Never write fake testimonials.
- Never feature "Case studies" or "Cases" navigation/page when no case studies exist (Stage 0-1). Use "Field notes" instead.
- Never feature "Pricing" page (canonical rule: AGS website has zero public pricing, CTA = qualification path).

### 5.2 Visual-related forbidden
- No em dash in cover graphics, slide titles, or any rendered text.
- No stock photography clichés (laptop + coffee + smiling team).
- No AI-generated faces or hands.
- No corporate gradient backgrounds.
- No "AI brand" cyan-purple-pink palette.
- No emoji-heavy slides.
- No decorative flourishes (curlicues, sparkles, glowing orbs).
- No fonts outside the four canonical (Playfair Display, DM Sans, Inter, JetBrains Mono).
- No colors outside the five canonical brand hexes (status signal colors for functional UI badges allowed as industry UX convention).
- No centered logo placement.
- No logo placement touching canvas edges (minimum 4-5% canvas dimension clear space).

## 6. APPLICATION EXAMPLES (reference outputs for each format)

### LinkedIn carousel slide (1080 wide x 1350 tall, PORTRAIT 4:5, updated v1.1)
```
ORIENTATION: PORTRAIT (vertical, taller than wide). 1080 px is the WIDTH (shorter dimension), 1350 px is the HEIGHT (longer dimension). Each slide is taller than wide. Same proportions as a smartphone screen held vertically. NEVER render as landscape.

Six canonical slide types: cover, stat, three-layer, pull-quote, steps, CTA close.
Standard carousel structure: cover + 3-5 body slides + CTA close (total 6-10 slides).
Per slide: one key idea, headline max 8 words, body max 30 words.
Alternate Soft Sandstone and Cosmic Navy backgrounds across body slides for texture variation.
Stat slides and pull-quote slides are high-impact pauses; alternate with three-layer and steps slides.
Keep maximum one Muted Gold note per slide.
AGS wordmark bottom right Inter SemiBold 22-28pt 60% opacity, with 4-5% canvas clear space.
Slide number bottom right Inter SemiBold (size proportional to wordmark) 50% opacity (format "01 / 07", no em dash).

TOPIC-SPECIFIC ICON PLACEMENT (added v1.2, Tomasz directive):
- Topic-specific icons illustrating the slide concept always positioned in LOWER ZONE of canvas, horizontally centered.
- Position approximately 75-80% canvas height from top.
- Clear space requirements: breathing room above (from content block) and below (from footer wordmark + domain + swipe row).
- No icons in upper half of canvas, never beside or overlapping headlines, eyebrows, or page indicators.
- Icon serves as visual anchor bridge between content block and footer.
```

### LinkedIn Article cover (1280x720 px, updated v1.1)
```
Four canonical variants: bone (Sandstone default), ink (Navy + gold accent), gold seal (Muted Gold solid), split (Sandstone + Navy).
Fifth variant when applicable: photo (Tomasz portrait, see section 3.4.1).
Headline does the work: Playfair Display Bold, max 8 words.
One-word italic flourish in Muted Gold (verb or noun, never adjective). Examples: "voice", "vibe", "leash", "architecture".
Construction recipe: eyebrow caps + headline + one circuit trace + byline + nothing else.
Logo clear space minimum 4-5% canvas dimension on all sides, never touch edges.
Large G monogram watermark (if used) may bleed off right or bottom edge, NEVER off top edge. Min 4-5% top clearance always.
AGS wordmark + reading time + authenticgrowthsystems.com url in bottom corners Inter SemiBold 22-28pt (NOT 14pt) 60% opacity, sized for thumbnail legibility.

PHOTO VARIANT (refined v1.2 per Tomasz directive): When cover features Tomasz photo, photo fills the ENTIRE canvas as full-bleed background. Text overlays the lower zone of the photo (sans-serif DM Sans Bold or Inter Bold, NOT Playfair). G monogram top-right Muted Gold, dotty pattern top-left, small avatar circle + name + tagline bottom-left. See section 3.4.1 for full rules. Side-panel photo placeholder is REJECTED anti-pattern.
```

### Sales deck slide (1920x1080 px)
```
Alternating: Soft Sandstone or Cosmic Navy.
One key fact per slide.
Headline Playfair Bold 56pt, body DM Sans 24pt.
Slide footer: section name + slide number + AGS wordmark, Inter SemiBold 12pt 50% opacity.
```

```
THREE STANDARD X DIMENSIONS (updated v1.2):
- Square 1080x1080 (1:1) — standalone X post graphic, max mobile feed footprint.
- Landscape 1200x675 (16:9) — when sharing a LinkedIn Article or any 16:9 cover, X reads same ratio.

Treatment:
High contrast Cosmic Navy or Soft Sandstone.

Body or supporting line DM Sans 24pt.
Numbers oversized when present.
```
